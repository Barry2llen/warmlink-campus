# 暖校 Link / WarmLink Campus

暖校 Link 是一个面向高校师生的 AI 校园互助与民生温度地图前端 Demo。项目围绕校园闲置物品共享、学习资料共享、生活物资互助、临时求助、公益帮扶、安全提醒和管理端数据治理，构建一套可演示的校园互助闭环。

当前版本以 Hackathon 路演和产品原型演示为目标，重点展示：

- 一句话 AI 智能发布互助需求
- 隐私信息检测与安全提醒
- 互助大厅搜索、筛选、响应
- 单条互助事项详情与一键帮助
- 校园温度地图和区域热力点位
- 个人暖心值、信用分、徽章和温度报告
- 管理端 Dashboard 与风险审核

## 项目定位

产品名称：暖校 Link  
英文名称：WarmLink Campus  
产品形态：Web 应用 / 移动端优先 / 可扩展为小程序  
适用场景：Trae On Campus AI Coding Hackathon  
对应赛题：邻里互助站 / 民生无界

一句话描述：

> 用户只需要输入一句自然语言需求，系统即可自动识别需求类型、生成互助事项、提示隐私与安全风险，并匹配合适资源或帮助者。

## 核心能力

### AI 智能发布

用户输入自然语言需求后，系统会生成结构化互助卡片：

- 自动识别事项类型：闲置物品、学习资料、生活物资、临时求助、公益帮扶、安全提醒
- 自动生成标题、描述、地点、时间要求、标签
- 判断紧急程度：高 / 中 / 低
- 检测隐私风险：手机号、微信号、QQ 号、学号、宿舍门牌号、银行卡号
- 生成安全提醒：公共区域交接、夜间求助、贵重物品归还约定等
- 展示推荐帮助者或资源
- 用户确认后发布到互助大厅和温度地图

当前 Demo 中 AI 解析使用本地规则模拟，位于 [src/App.tsx](src/App.tsx) 的 `analyzeDraft` 逻辑；隐私检测为独立模块，位于 [src/lib/privacy/detect.ts](src/lib/privacy/detect.ts)。项目也保留了 LLM 输出 schema，位于 [src/lib/llm/schema.ts](src/lib/llm/schema.ts)，方便后续接入真实模型服务。

### 互助大厅

互助大厅用于集中展示校园内正在发生的互助事项：

- 按类型筛选
- 关键词搜索
- 排序入口
- 查看事项卡片
- 进入详情页
- 一键发送帮助意向
- 展示发布者信用分、暖心值、响应人数、状态和标签

### 校园温度地图

温度地图用于可视化展示校园互助热点：

- 模拟校园空间区域：图书馆、宿舍区、一食堂、教学楼、快递站、体育场
- 展示区域今日互助次数、活跃事项、平均响应时间、温度指数
- 点击点位查看区域详情
- 从区域进入区域互助页
- 从区域直接发起求助

### 信用与激励

个人中心展示用户互助贡献：

- 暖心值
- 信用分
- 本周互助数据
- 徽章预览
- 我的发布 / 我的帮助 / 我的收藏
- 个人温度报告

### 管理端治理

管理端用于展示平台运行与风险治理能力：

- 今日新增求助
- 今日完成互助
- 平均响应时间
- 活跃用户数
- 风险提醒数
- 用户满意度
- 互助类型分布
- 区域热力排行
- 实时风险提醒
- AI 治理建议
- 风险审核与处理操作

## 环境变量

项目提供 `.env.example`：

```bash
VITE_GEMINI_API_KEY=your-key-here
```

复制为本地环境文件：

```bash
cp .env.example .env.local
```

Windows PowerShell 可使用：

```powershell
Copy-Item .env.example .env.local
```

说明：

- 当前前端 Demo 默认不依赖真实 Gemini API Key。
- `VITE_GEMINI_API_KEY` 是为后续接入真实 LLM 服务预留。
- 若接入真实模型，生产环境不应在前端直接暴露 API Key，应改为后端代理或 Serverless API。

## 推荐演示路径

### 核心闭环演示

1. 打开首页 `/`
2. 在首页输入一句需求，例如：

   ```text
   我在图书馆想借一把伞，20 分钟后回南区宿舍
   ```

3. 点击 `AI 发布`
4. 进入 `/publish`
5. 点击 `AI 生成互助卡片`
6. 查看 AI 解析结果、隐私提醒、安全提醒和推荐帮助者
7. 点击 `确认发布`
8. 进入 `/publish/success`
9. 点击 `去互助大厅看看`
10. 在 `/hall` 搜索或筛选事项
11. 点击事项进入 `/help/:id`
12. 点击 `一键帮助`
13. 进入 `/map` 查看校园温度地图
14. 进入 `/profile` 查看个人暖心值和信用沉淀

### 管理端演示

1. 进入 `/admin`
2. 查看核心指标、互助类型分布、区域热力排行
3. 查看 AI 治理建议
4. 点击进入 `/admin/risk`
5. 查看风险详情与处理操作

### 隐私检测演示输入

可以在 `/publish` 输入以下内容验证隐私检测：

```text
今晚 21:30 想找同学从第三教学楼结伴回宿舍，微信 wxCampus88
```

或：

```text
我在 8 号楼 603 室，手机号 13812345678，想借一把伞
```

系统会识别联系方式或宿舍门牌等敏感信息，并在发布描述中进行脱敏提示。

## 数据与状态说明

当前项目为纯前端 Demo，数据保存在 React 本地状态中。

已模拟的数据包括：

- 用户列表
- 校园区域列表
- 互助事项列表
- 风险提醒列表
- 推荐帮助者列表

当前不会持久化到后端或本地存储。刷新页面后，运行时新增的发布事项和响应状态会恢复为初始 Mock 数据。

## AI 接入说明

当前实现分为两层：

1. 可演示层：`analyzeDraft` 使用本地规则完成分类、标题、描述、紧急程度、地点、标签和安全提醒。
2. 扩展层：`LLM_RESPONSE_JSON_SCHEMA` 已定义真实模型输出结构，后续可替换为 Gemini、OpenAI 或校内部署模型。

建议后续真实接入方式：

```text
前端输入
  -> 后端 API / Serverless API
  -> LLM 服务
  -> JSON Schema 校验
  -> 隐私检测二次校验
  -> 返回结构化草稿
  -> 用户确认发布
```

不要在生产环境中直接让浏览器调用模型供应商 API。

## 当前限制

当前版本是前端原型，不包含：

- 真实登录认证
- 后端数据库
- 真实文件上传下载
- 真实即时通讯
- 真实地图服务
- 真实支付或押金
- 真实管理员权限控制
- 真实模型服务调用
- 多端数据持久化

这些能力可以作为后续迭代方向。

## 后续迭代方向
